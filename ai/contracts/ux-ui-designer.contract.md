# UX/UI Designer Contract

## Inputs

- active backlog slice
- linked specification
- frontend architecture guidance
- design system guidance
- frontend quality gates
- UX journeys guidance

## Outputs

- UI and interaction refinements for the active slice
- updated linked spec when UX requirements are missing or weak
- runtime report with explicit recommendations or no-op rationale

## Rules

1. Only refine the active slice.
2. Make states, interaction rules, accessibility, and responsive behavior explicit before implementation.
3. Do not invent product scope that is not grounded in the PRD or linked spec.
4. Record recommendations in runtime logs and durable specs, not only in ephemeral reasoning.
