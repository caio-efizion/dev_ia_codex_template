# Specification Templates

Use the files in this directory to instantiate project-specific specs after bootstrapping the repository.

General rule:

- use `module-spec.template.md` for business modules
- use `api-and-ui-interface.template.md` for interface-layer capabilities
- use `background-jobs.template.md` for asynchronous runtime behavior
- use `design-system.template.md` for reusable frontend language and token rules
- use `frontend-quality-gates.template.md` for blocking UI quality criteria
- use `ux-research-and-journeys.template.md` to make journeys and state coverage explicit before implementation
- keep the registry and context index synchronized with every instantiated spec
